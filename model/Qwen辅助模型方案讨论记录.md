# Qwen辅助模型方案讨论记录

## 背景
需要创建一个辅助模型来解决Qwen大模型识别民族服饰时可能识别错误的问题。问题的核心是：有些民族服饰外表相似，真正的区分在于细节处。

## 初始方案（已废弃）

### 方案：级联架构
```
图像 → Qwen初步识别 → 辅助模型细节验证 → 最终结果
```
**问题**：重新搜索还是依赖大模型，可能导致重复错误。

### 方案：并行融合架构
```
图像 → Qwen识别
     → 辅助模型细节提取
     → 结果融合 → 最终结果
```
**问题**：计算开销大。

### 方案：特征增强架构
```
图像 → 辅助模型提取细节特征 → 增强后的图像 → Qwen识别
```
**问题**：特征提取质量难以保证。

## 改进方案（已讨论）

### 方案：细节特征文字描述共享
```
原图 → 辅助模型提取细微特征 → 文本描述 → Qwen识别
```

**辅助模型输出格式**：
```json
{
  "embroidery_style": "十字绣",
  "color_palette": {
    "primary": "黑色",
    "secondary": "红色"
  },
  "pattern_type": "几何纹样",
  "craftsmanship": ["蜡染", "挑花"]
}
```

**问题**：文字描述无法准确传达视觉特征的细微差异。

## 最终方案：视觉细节增强（已确认）

### 核心架构
```
原图 → 辅助模型定位关键细节区域
→ 裁剪并放大这些区域（6-8张）
→ 原图 + 细节放大图 → Qwen识别
```

### 辅助模型的作用
**只做一件事：找到关键区域**

```python
# 辅助模型输出（极简）：
{
  "regions": [
    {"id": 1, "type": "neckline", "bbox": [x1, y1, x2, y2]},
    {"id": 2, "type": "embroidery", "bbox": [x1, y1, x2, y2]},
    {"id": 3, "type": "silver", "bbox": [x1, y1, x2, y2]},
    {"id": 4, "type": "pattern", "bbox": [x1, y1, x2, y2]},
    {"id": 5, "type": "sleeve", "bbox": [x1, y1, x2, y2]},
    {"id": 6, "type": "waistband", "bbox": [x1, y1, x2, y2]}
  ]
}
```

### 关键区域类型
1. **领口（neckline）**：立领、圆领、无领等
2. **刺绣（embroidery）**：刺绣图案区域
3. **银饰（silver）**：银饰装饰区域
4. **纹样（pattern）**：图案纹理区域
5. **袖口（sleeve）**：袖口装饰
6. **腰带（waistband）**：腰带设计

### 为什么这个方案可行？

1. **不依赖文字描述**：让Qwen自己"看"细节
2. **计算可控**：裁剪图不需要额外推理
3. **利用Qwen优势**：多模态模型擅长图像理解
4. **无需参考图库**：不需要预先收集标准特征图
5. **数据需求少**：只需标注边界框，不需要特征描述

## 训练数据需求

### 数据准备
- **图像来源**：现有 `56个少数民族图片` 和 `6大民族详细图片`
- **标注方式**：在图像上框出关键区域
- **标注类型**：6-8种关键区域类型

### 数量要求
- 每个民族：20-50张图像
- 每张图像：标注5-8个区域
- 总计：1000-2000个标注框

### 标注内容
**不需要标注"这是什么"**，只需要标注"这个区域重要"：
- 例如：框出刺绣区域，不需要说"这是十字绣"
- 辅助模型只负责"找"，不负责"识别"

## 技术栈选择

### 辅助检测模型
- **选择**：YOLOv8（轻量、快速）
- **任务**：目标检测（检测关键区域）
- **输出**：边界框 + 区域类型

### 细节处理
- **裁剪**：根据bbox裁剪图像
- **放大**：4倍放大细节
- **组合**：原图 + 6-8张细节图

### 与Qwen集成
- **输入**：多图（原图 + 细节图）
- **API**：Qwen多模态API
- **输出**：民族识别结果
